Minha Biblioteca

Sobre

API desenvolvida para integrar a biblioteca online com qualquer sistema.

Os endpoints recebem a requisição com as informações necessárias no body.

A API processa os dados e envia para a biblioteca online verificar.

Caso a biblioteca retorne positivo, a URL autenticada é devolvida ao sistema requerente.

Construção

1 - Os endpoints foram criados na pasta "functions" do app Netlify do Grupo Nobre.

2 - As variáveis de ambientes devem ser criadas via comando "ntl env:set" ou importadas por "ntl env:import". (Para o import as variáveis devem estar no arquivo .env na raiz do projeto)

2.1 - Instale os pacotes necessários com npm.

3 - Crie o arquivo netlify.toml na raiz do projeto passando:
[build]
functions = "functions"
publish = "public"

4 - Crie uma pasta "functions" para construção do backend da aplicação.

4.1 - Os endpoints da API são acessados por meio de "/.netlify/function/nome_da_funcao".

5 - Crie uma pasta "public" para construção do frontend da aplicação.

Aplicação

1 - "ntl help" revela todos os comandos do netlify.

2 - "ntl comando help" mostra como usar este determinado comando.

3 - Mais usados:
a) "ntl init" - Incializa o projeto.
a) "ntl login" - Conecta o projeto à conta Netlify.
b) "ntl dev" - Gera um ambiente de desenvolvimento.
c) "ntl env" - Manipulação das variáveis de ambiente.
d) "ntl status" - Verifica informações importantes.
e) "ntl functions" - Informa o status dos endpoints.
f) "ntl deploy" - Testa o deploy.
g) "ntl deploy --prod" - Sobe o projeto para a web.
